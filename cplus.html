C++ Programming Guide

BY: MUHAMMAD BILAL

Introduction to C++

C++ is a powerful, versatile programming language that extends the C language by adding object-oriented features. It is widely used in software development, game programming, and systems programming.

Beginner Level

Introduction to C++

C++ is an extension of the C programming language that includes object-oriented features. It's a popular choice for system/software development and competitive programming.

History of C++

C++ was developed by Bjarne Stroustrup at Bell Labs in the early 1980s as an enhancement to C. It added object-oriented programming features to the existing C language.

Features of C++

  • Object-Oriented Programming
  • Memory Management
  • Multiple Inheritance
  • Function Overloading

Writing the First C++ Program

The basic structure of a C++ program includes headers, the main function, and statements. Here's a simple example:

#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}

Intermediate Level

Object-Oriented Programming (OOP)

OOP allows developers to model real-world entities using classes and objects, providing a way to organize and manage code more efficiently.

  • Classes and objects: Basics of creating and using classes and objects
  • Access specifiers: public, private, protected
  • Constructors and destructors: Special functions for initialization and cleanup
  • Member functions: Functions that operate on class members
  • Static members: Class members that are shared among all instances
  • This pointer: A pointer to the current object
  • Friend functions and classes: Functions or classes that can access private members
#include <iostream>

class Rectangle {
public:
    int width, height;

    Rectangle(int w, int h) : width(w), height(h) {}

    int area() {
        return width * height;
    }
};

int main() {
    Rectangle rect(10, 5);
    std::cout << "Area: " << rect.area() << std::endl;
    return 0;
}

Inheritance

Inheritance allows one class (derived class) to inherit properties and methods from another class (base class).

  • Base and derived classes: Basic inheritance model
  • Types of inheritance: Single, multiple, multilevel, hierarchical, hybrid
  • Constructor and destructor in inheritance: Handling constructors and destructors in derived classes
  • Function overriding: Redefining base class methods in derived classes
#include <iostream>

class Base {
public:
    void show() {
        std::cout << "Base class" << std::endl;
    }
};

class Derived : public Base {
public:
    void show() {
        std::cout << "Derived class" << std::endl;
    }
};

int main() {
    Derived obj;
    obj.show();
    return 0;
}

Polymorphism

Polymorphism allows objects to be treated as instances of their base class rather than their actual class, enabling method overriding.

  • Function overloading: Multiple functions with the same name but different parameters
  • Operator overloading: Customizing the behavior of operators
  • Virtual functions: Functions that can be overridden in derived classes
  • Pure virtual functions and abstract classes: Defining abstract base classes
#include <iostream>

class Base {
public:
    virtual void show() {
        std::cout << "Base class" << std::endl;
    }
};

class Derived : public Base {
public:
    void show() override {
        std::cout << "Derived class" << std::endl;
    }
};

int main() {
    Base *basePtr;
    Derived derivedObj;
    basePtr = &derivedObj;
    basePtr->show();
    return 0;
}

Templates

Templates enable writing generic and reusable code by allowing functions and classes to operate with different data types.

  • Function templates: Defining generic functions
  • Class templates: Defining generic classes
  • Template specialization: Customizing template behavior for specific types
#include <iostream>

template <typename T>
T max(T a, T b) {
    return (a > b) ? a : b;
}

int main() {
    std::cout << max(10, 20) << std::endl;
    std::cout << max(10.5, 5.5) << std::endl;
    return 0;
}

Exception Handling

Exception handling provides a way to manage errors and exceptional situations in a controlled manner using try, catch, and throw statements.

  • Try, catch, and throw statements: Syntax and usage
  • Standard exceptions: Built-in exception classes
  • Custom exceptions: Defining and using custom exception classes
#include <iostream>

int divide(int a, int b) {
    if (b == 0) throw std::invalid_argument("Division by zero");
    return a / b;
}

int main() {
    try {
        std::cout << divide(10, 2) << std::endl;
        std::cout << divide(10, 0) << std::endl;
    } catch (const std::invalid_argument &e) {
        std::cerr << "Error: " << e.what() << std::endl;
    }
    return 0;
}

  • Iterators: Traversing containers

Advanced Data Structures

Advanced data structures are crucial for optimizing complex algorithms and managing large amounts of data efficiently.

  • Trees: Binary tree, binary search tree, AVL tree, etc.
  • Graphs: Representation, traversal algorithms

class Node {

Advanced Algorithms

Advanced algorithms are used to solve complex problems efficiently, including sorting, searching, and optimization techniques.

  • Sorting algorithms: Quick sort, merge sort, heap sort, etc.
  • Searching algorithms: Binary search, hash-based search
  • Dynamic programming: Problem-solving approach for optimization
  • Greedy algorithms: Algorithms that make locally optimal choices
  • Backtracking: Algorithmic technique for solving problems incrementally
#include <iostream>

void quickSort(int arr[], int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}

int partition(int arr[], int low, int high) {
    int pivot = arr[high];
    int i = (low - 1);

    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            std::swap(arr[i], arr[j]);
        }
    }
    std::swap(arr[i + 1], arr[high]);
    return (i + 1);
}

int main() {
    int arr[] = {10, 7, 8, 9, 1, 5};
    int n = sizeof(arr) / sizeof(arr[0]);
    quickSort(arr, 0, n - 1);
    std::cout << "Sorted array: ";
    for (int i = 0; i < n; i++) {
        std::cout << arr[i] << " ";
    }
    std::cout << std::endl;
    return 0;
}

Multithreading and Concurrency

Multithreading and concurrency involve executing multiple threads simultaneously to improve performance and resource utilization.

  • Thread creation and management: Creating and managing threads
  • Synchronization: Using mutex, lock, condition variables
  • Thread communication: Mechanisms for communication between threads
#include <iostream>
#include <thread>

void printHello() {
    std::cout << "Hello from thread!" << std::endl;
}

int main() {
    std::thread t1(printHello);
    t1.join();
    return 0;
}

File Handling

File handling involves reading from and writing to files using file streams.

  • File streams: ifstream, ofstream, fstream
  • Reading from and writing to files: Basic file operations
  • Binary file operations: Working with binary files
#include <iostream>
#include <fstream>

int main() {
    std::ofstream outFile("example.txt");
    outFile << "Hello, file!" << std::endl;
    outFile.close();

    std::ifstream inFile("example.txt");
    std::string line;
    if (getline(inFile, line)) {
        std::cout << line << std::endl;

    inFile.close();

    return 0;
}

Design Patterns

Design patterns are reusable solutions to common problems in software design.

  • Creational patterns: Singleton, Factory, Abstract Factory, Builder, Prototype
  • Structural patterns: Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy
  • Behavioral patterns: Chain of Responsibility, Command, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor
#include <iostream>

class Singleton {
private:
    Singleton() {}
    static Singleton *instance;

public:
    static Singleton *getInstance() {
        if (!instance) {
            instance = new Singleton();
        }
        return instance;
    }
};

Singleton *Singleton::instance = nullptr;

int main() {
    Singleton *singleton = Singleton::getInstance();
    std::cout << "Singleton instance address: " << singleton << std::endl;
    return 0;
}
